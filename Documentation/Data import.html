Data import

PSB_GUI enables the semi-automatic import of the experimental data. These should be provided in the tabulated structure in a text file or spread sheet. The example of the data format is shown in the following figure:

To refine peak shift (PS) and peak broadening (PB) the following data are mandatory:

  • H, K, L diffraction indices - for both PS and PB
  • the positions of the Bragg reflections (2θ, or q) - for both PS and PB
  • ψ angle (inclination with respect to the sample normal) - for PS
  • either FWHM or broadening values (or both) - for PB

Other parameters are optional, also depending on the fact if you want to use some advanced models for the refinement. The information about the wavelength should be included at the beginning of the input file before the names of individual columns. However, it is not mandatory and could be also specified additionaly via Manual selection tool.

If the columns are properly named, the input script tries to import the corresponding data. If the search for individual columns went wrong, the corresponding cells in the Data table would be filled with NaNs. However, the corresponding data still may be extracted by the Manual selection.

Manual selection

This is the overview of the Manual selection window:

On the left, you can see the preview of the data available after the input, together with the recognized column names (the strings after var#:). On the right hand side, you have several options for the choice of corresponding data. You can select the corresponding column together with the appropriate unit, or you can compute the corresponding data using a formula written in Matlab syntax. Please note, that the native units for all angular values in this program are degrees.

In the example of the abovementioned figure, the measured data were fitted by an asymmetric pseudo-Voigt function during the preprocessing. Therefore, the input data included fwhm on the left and right side of the peak. The fwhm used then for PSB_GUI is specified by the formula: (var8+var9)/2, which means it will be computed as the average of the columns 8 and 9 from the table on the left.

The wavelength could be specified by a custom number or from the list of predefined values.
